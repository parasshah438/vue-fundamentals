What is Next.js?

Next.js is a React framework for building full-stack web applications. You use React Components to build user interfaces, and Next.js for additional features and optimizations. It automatically configures bundlers and compilers so you can focus on building your product.

Next.js is a powerful React framework for building fast, scalable, and SEO-friendly web applications. It provides built-in features like server-side rendering, static site generation, routing, and API handling.

Built on React

Easier to create modern, component-driven UIs with all React ecosystem tools.

SSR & SSG

Pages pre-rendered on the server — improving SEO and initial load performance.

Auto Optimized

Automatic bundling, code splitting, and asset optimization out of the box.

Enterprise Ready

Suitable for small projects and enterprise-grade applications with full-stack support.

Think of it this way
React

A library for building UI components (like buttons, forms, lists)

Routing in Next.js

Next.js routing is a built-in, file-based system that automatically maps URLs to files and folders — no extra libraries needed.

File-Based

No config, just create files/folders in app/

Dynamic & Nested

Folder naming supports nested and variable routes

Fast Navigation

Built-in Link component with preloading

SEO URLs

Redirects, SEO-friendly URLs, and optimized performance by default

Core concept: Routing = which page/code runs when a user visits a URL.

Types of Routes

1Static Route — Fixed URL
src/app/about/page.tsx       →  /about
2Nested Route — Sub-pages
src/app/dashboard/page.tsx            →  /dashboard
src/app/dashboard/settings/page.tsx   →  /dashboard/settings
src/app/dashboard/profile/page.tsx    →  /dashboard/profile
3Dynamic Route — Variable in URL
Wrap the folder name in square brackets [param] to create a dynamic segment. 
src/app/blog/[slug]/page.tsx    →  /blog/hello-world
                                →  /blog/my-post
                                →  /blog/anything

src/app/users/[id]/page.tsx     →  /users/1
                                →  /users/42
                                →  /users/999
4Catch-All Route — Multiple Segments
src/app/docs/[...slug]/page.tsx   →  /docs/intro
                                  →  /docs/react/hooks
                                  →  /docs/a/b/c/d
5Optional Catch-All — With or Without Segments
src/app/shop/[[...filters]]/page.tsx  →  /shop            (no filters)
                                      →  /shop/men
                                      →  /shop/men/shoes
6Route Groups — Organise Without Affecting URL
Wrap folder name in parentheses (group) — the group name is not added to the URL. 
src/app/(marketing)/page.tsx           →  /
src/app/(marketing)/about/page.tsx     →  /about
src/app/(dashboard)/home/page.tsx      →  /home
src/app/(dashboard)/settings/page.tsx  →  /settings
7API Route — Backend Endpoint (No UI)
File must be named route.ts (not page.tsx) to create an API endpoint. 
src/app/api/users/route.ts          →  /api/users        (GET, POST, etc.)
src/app/api/users/[id]/route.ts     →  /api/users/42
src/app/api/posts/route.ts          →  /api/posts

Server vs Client Components

In Next.js (App Router), every component is a React Server Component (RSC) by default. Add "use client" at the top to opt into Client Components.

Server Component Default
  • Runs only on the server — never sent to browser
  • Can directly call databases, APIs, file system
  • Reduces JavaScript bundle size
  • Cannot use hooks (useState, useEffect)
  • Cannot use browser APIs (window, localStorage)
  • Cannot have onClick / event handlers
// app/posts/page.tsx  (Server Component — default)
async function PostsPage() {
  const res  = await fetch('https://api.example.com/posts');
  const data = await res.json();

  return (
    <ul>
      {data.map((p: any) => (
        <li key={p.id}>{p.title}</li>
      ))}
    </ul>
  );
}
export default PostsPage;
Client Component
  • Runs in the browser (and server for initial HTML)
  • Can use React hooks (useState, useEffect)
  • Can use browser APIs & event listeners
  • Cannot directly access server-only resources
  • Adds JavaScript to the browser bundle
'use client'; // ← Must be first line

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(c => c + 1)}>
      Count: {count}
    </button>
  );
}
Quick Decision Guide
Need this?Use
Fetch data from DB / APIServer Component
Access backend secrets / tokensServer Component
useState / useEffectClient Component
Click / change event handlersClient Component
Browser APIs (window, navigator)Client Component
Static UI with no interactivityServer Component
Best Practice: Keep Server Components as the default. Only add "use client" to the smallest leaf components that actually need interactivity. Use Client Components as leaves, not roots.

Layouts & Special Files

Next.js App Router has a set of reserved file names that give each folder special behavior — no configuration needed.

layout.tsx

Shared UI wrapper that persists across navigations. Does not re-render on route change. Perfect for nav, footer, sidebar.

page.tsx

The actual route UI — only page.tsx is publicly accessible via URL. Required in every route folder.

loading.tsx

Automatically shown as a Suspense fallback while the page is loading. No extra code needed.

error.tsx

Error boundary UI — shown when the page throws. Must be a "use client" component. Receives error and reset props.

not-found.tsx

Rendered when notFound() is called or a route doesn't exist. Custom 404 page.

template.tsx

Like layout.tsx but re-renders on every navigation. Useful for animations or per-route analytics.

route.ts

API endpoint handler file. Exports GET, POST, PUT, DELETE functions. Not accessible as a page.

default.tsx

Fallback UI for Parallel Routes when a slot has no active match.

Root Layout (Required)
// app/layout.tsx  — root layout wraps every page
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <Header />
        <main>{children}</main>
        <Footer />
      </body>
    </html>
  );
}

Data Fetching (SSR / SSG / ISR)

In the App Router, data fetching happens directly inside async Server Components using the native fetch() API extended by Next.js.

SSR

Fetched fresh on every request. Always up-to-date.

cache: 'no-store'
SSG

Fetched once at build time. Fastest, served from CDN.

cache: 'force-cache'
ISR

Cached but re-validated after N seconds. Best of both worlds.

next: { revalidate: 60 }
1SSR — Server-Side Rendering (Dynamic)
// Fetched fresh on EVERY request
async function UsersPage() {
  const res  = await fetch('https://api.example.com/users', {
    cache: 'no-store',   // ← SSR: never cache
  });
  const users = await res.json();

  return <ul>{users.map((u: any) => <li key={u.id}>{u.name}</li>)}</ul>;
}
2SSG — Static Site Generation (Build Time)
// Generated ONCE at build time, served from CDN
async function BlogPage() {
  const res  = await fetch('https://api.example.com/posts', {
    cache: 'force-cache',  // ← SSG: cache forever
  });
  const posts = await res.json();

  return <ul>{posts.map((p: any) => <li key={p.id}>{p.title}</li>)}</ul>;
}
3ISR — Incremental Static Regeneration
// Re-validated every 60 seconds in background
async function NewsPage() {
  const res  = await fetch('https://api.example.com/news', {
    next: { revalidate: 60 },  // ← ISR: refresh every 60s
  });
  const news = await res.json();

  return <ul>{news.map((n: any) => <li key={n.id}>{n.headline}</li>)}</ul>;
}

// Optional: set at segment level
export const revalidate = 60;
4generateStaticParams — Pre-build Dynamic Routes
// app/blog/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = await fetch('https://api.example.com/posts').then(r => r.json());
  return posts.map((p: any) => ({ slug: p.slug }));
}

export default async function Post({ params }: { params: { slug: string } }) {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`)
    .then(r => r.json());
  return <article><h1>{post.title}</h1><p>{post.body}</p></article>;
}

Server Actions

Server Actions let you run server-side code directly from forms and components — no separate API route needed. They are async functions marked with "use server".

Secure

Code never reaches client bundle — runs only on server

No API Route Needed

Mutations directly from components without fetch() calls

Progressive Enhancement

Forms work even before JavaScript loads

1Server Action in a Server Component
// app/contact/page.tsx
export default function ContactPage() {
  async function handleSubmit(formData: FormData) {
    'use server'; // ← marks this as a Server Action

    const name  = formData.get('name');
    const email = formData.get('email');
    // Save to DB, send email, etc.
    console.log('Received:', name, email);

  return (
    <form action={handleSubmit}>
      <input name="name"  placeholder="Name" />
      <input name="email" placeholder="Email" />
      <button type="submit">Send</button>
    </form>
  );
}

Navigate programmatically inside Client Components.

'use client';
import { useRouter }    from 'next/navigation';
import { usePathname }  from 'next/navigation';
import { useSearchParams } from 'next/navigation';

export default function MyComponent() {

Middleware

Middleware runs before a request is completed. It intercepts requests at the Edge, allowing you to rewrite, redirect, add headers, or check auth — before the page loads.

Auth Guard

Redirect unauthenticated users to login

A/B Testing

Rewrite to different pages based on cookies

i18n Routing

Detect locale and redirect to correct path

Rate Limiting

Block or throttle requests at the edge

middleware.ts — File Location
Place middleware.ts at the root of your project (same level as app/), not inside app/. 
// middleware.ts  (root level)
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(request: NextRequest) {
  const token = request.cookies.get('auth-token');

  // Protect /dashboard routes
  if (request.nextUrl.pathname.startsWith('/dashboard')) {
    if (!token) {
      return NextResponse.redirect(new URL('/login', request.url));
    }
  }

  // Add custom header
  const response = NextResponse.next();
  response.headers.set('x-custom-header', 'my-value');
  return response;
}

// Define which paths this middleware runs on
export const config = {
  matcher: ['/dashboard/:path*', '/profile/:path*'],
};

1Context API — Provider Pattern

In App Router, Context Providers must be Client Components wrapped around the layout.

// context/ThemeContext.tsx
'use client';
import { createContext, useContext, useState } from 'react';

type Theme = 'light' | 'dark';
const ThemeCtx = createContext<{ theme: Theme; toggle: () => void } | null>(null);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<Theme>('dark');
  return (
    <ThemeCtx.Provider value={{ theme, toggle: () => setTheme(t => t === 'dark' ? 'light' : 'dark') }}>
      {children}
    </ThemeCtx.Provider>
  );
}

export const useTheme = () => {
  const ctx = useContext(ThemeCtx);
  if (!ctx) throw new Error('useTheme must be inside ThemeProvider');
  return ctx;
};

// app/layout.tsx — wrap layout
import { ThemeProvider } from '@/context/ThemeContext';
export default function RootLayout({ children }: any) {
  return (
    <html><body>
      <ThemeProvider>{children}</ThemeProvider>
    </body></html>
  );
}

Security Best Practices

Input Validation

Always validate and sanitize user input with Zod — never trust raw form data.

Never Expose Secrets

Only use NEXT_PUBLIC_ for non-sensitive values. API keys / DB URLs stay server-only.

Verify Auth in Server Actions

Next.js uses React <Suspense> to stream HTML progressively to the browser. Slow data-fetching components show a fallback skeleton while loading, without blocking the rest of the page.

Faster TTFB

Static shell sent instantly, dynamic parts streamed in

Better UX
