Installation & Setup

System Requirements Windows
Minimum Requirements:
• Windows 10 64-bit: Home or Pro 21H1 (build 19043) or higher, or Enterprise or Education 20H2 (build 19042) or higher
• Windows 11 64-bit: Home or Pro version 21H2 or higher, or Enterprise or Education version 21H2 or higher
• Hardware: SLAT-capable 64-bit processor, 4GB RAM, UEFI firmware with Secure Boot capability
• Hyper-V and Containers: Windows features must be enabled
• WSL 2 backend: Windows Subsystem for Linux version 2 (recommended)

Hardware Virtualization:
• Virtualization must be enabled in BIOS/UEFI
• Hyper-V compatible processor required

Manual Installation Steps
Step 1: Enable Required Windows Features
1. Open "Turn Windows features on or off"
2. Check "Hyper-V" (all sub-features)
3. Check "Containers"
4. Check "Windows Subsystem for Linux" (for WSL 2)
5. Check "Virtual Machine Platform"
6. Restart your computer

Step 2: Install WSL 2 (Recommended Backend)
1. Download WSL 2 Linux kernel update
2. Run the installer
3. Set WSL 2 as default: wsl --set-default-version 2

Step 3: Download and Install Docker Desktop
1. Download Docker Desktop from official website
2. Run installer as Administrator
3. Choose "Use WSL 2 instead of Hyper-V" during installation
4. Complete installation and restart

Docker Architecture on Windows
Windows Docker Architecture:

Docker Desktop Components:
• Docker Engine: Runs in a Linux VM (via WSL2) or Hyper-V
• Docker Desktop: GUI application for managing Docker
• Docker CLI: Command-line interface accessible from Windows
• Container Runtime: containerd for Linux containers, Windows containers for Windows

Two Container Types on Windows:
• Linux Containers: Run via WSL2 or Hyper-V (default)
• Windows Containers: Native Windows containers using Windows kernel

Backend Options:
• WSL2 Backend: Recommended, better performance, file sharing
• Hyper-V Backend: Traditional virtualization, more isolated

File System Integration:
• Windows drives mounted in WSL2 at /mnt/c, /mnt/d, etc.
• Linux filesystem accessible from Windows at \\wsl$\

Dockerfile Instructions
# FROM - Base image
FROM node:16-alpine

# LABEL - Metadata
LABEL version="1.0" description="My Node.js app"

# ENV - Environment variables
ENV NODE_ENV=production
ENV PORT=3000

# WORKDIR - Working directory
WORKDIR /usr/src/app

# COPY vs ADD
COPY package*.json ./        # Copy specific files
ADD https://example.com/file.tar.gz /tmp/  # Can download and extract

# RUN - Execute commands during build
RUN npm install --only=production

# USER - Switch user
USER node

# VOLUME - Mount points
VOLUME ["/data"]

# EXPOSE - Document ports
EXPOSE $PORT

# CMD vs ENTRYPOINT
CMD ["node", "server.js"]    # Can be overridden
ENTRYPOINT ["node"]          # Always executed
CMD ["server.js"]            # Default arguments for ENTRYPOINT

Windows File Sharing Configuration
Docker Desktop File Sharing Settings:

Configuring Drive Access:
1. Open Docker Desktop → Settings → Resources → File Sharing
2. Add drives/paths you want to share with containers
3. Common paths: C:\, D:\, C:\Users\, C:\Development\
4. Apply & Restart Docker Desktop

WSL2 File Sharing:
• Windows drives automatically mounted at /mnt/c, /mnt/d
• Better performance for Linux containers
• Access WSL2 files from Windows: \\wsl$\Ubuntu\home\user\

Performance Considerations:
• Cross-OS file access (Windows↔Linux) is slower
• Store files in WSL2 filesystem for Linux containers
• Use Windows filesystem for Windows containers

# PowerShell script for Docker health monitoring
function Check-DockerHealth {
    param(
        [string]$ContainerName
    )
    
    $containerStatus = docker inspect $ContainerName --format '{{.State.Health.Status}}' 2>$null
    
    if ($containerStatus -eq "healthy") {
        Write-Host "$ContainerName is healthy" -ForegroundColor Green
    } elseif ($containerStatus -eq "unhealthy") {
        Write-Host "$ContainerName is unhealthy" -ForegroundColor Red
        docker logs $ContainerName --tail 10
    } else {
        Write-Host "$ContainerName health status unknown" -ForegroundColor Yellow
    }
}

# PowerShell Docker utility functions

# Function to build and tag Docker image
function Build-DockerImage {
    param(
        [string]$ImageName,
        [string]$Tag = "latest",
        [string]$DockerfilePath = "."
    )
    
    $fullImageName = "${ImageName}:${Tag}"
    Write-Host "Building Docker image: $fullImageName" -ForegroundColor Cyan
    
    docker build -t $fullImageName $DockerfilePath
    
    if ($LASTEXITCODE -eq 0) {
        Write-Host "Build successful: $fullImageName" -ForegroundColor Green
    } else {
        Write-Host "Build failed!" -ForegroundColor Red
    }
}

# Function to run container with common options
function Run-DockerContainer {
    param(
        [string]$ImageName,
        [string]$ContainerName,
        [hashtable]$Ports = @{},
        [hashtable]$Volumes = @{},
        [hashtable]$Environment = @{}
    )
    
    $dockerArgs = @("run", "-d", "--name", $ContainerName)
    
    # Add port mappings
    foreach ($port in $Ports.GetEnumerator()) {
        $dockerArgs += "-p", "$($port.Key):$($port.Value)"
    }
    
    # Add volume mounts
    foreach ($volume in $Volumes.GetEnumerator()) {
        $dockerArgs += "-v", "$($volume.Key):$($volume.Value)"
    }
    
    # Add environment variables
    foreach ($env in $Environment.GetEnumerator()) {
        $dockerArgs += "-e", "$($env.Key)=$($env.Value)"
    }
    
    $dockerArgs += $ImageName
    
    Write-Host "Running: docker $($dockerArgs -join ' ')" -ForegroundColor Cyan
    & docker @dockerArgs
}

# Example usage
Run-DockerContainer -ImageName "nginx:latest" -ContainerName "my-nginx" -Ports @{"8080"="80"}

# PowerShell script for cross-platform Docker builds
function Build-MultiPlatform {
    param(
        [string]$ImageName,
        [string]$Tag = "latest"
    )
    
    # Create buildx builder if it doesn't exist
    $builderName = "multiplatform"
    $existingBuilder = docker buildx ls | Select-String $builderName
    
    if (-not $existingBuilder) {
        Write-Host "Creating multi-platform builder..." -ForegroundColor Cyan
        docker buildx create --name $builderName --use
        docker buildx inspect --bootstrap
    } else {
        docker buildx use $builderName
    }
    
    # Build for multiple platforms
    $platforms = "linux/amd64,linux/arm64,windows/amd64"
    Write-Host "Building for platforms: $platforms" -ForegroundColor Cyan
    
    docker buildx build --platform $platforms -t "${ImageName}:${Tag}" --push .
}

# Example: Build multi-platform image
Build-MultiPlatform -ImageName "myregistry/myapp" -Tag "v1.0"

# Alternative: Build specific platform
