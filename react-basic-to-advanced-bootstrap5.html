What is React?

React is a JavaScript library for building user interfaces using reusable components. You describe the UI for each state of your app, and React updates the DOM efficiently when data changes.

React is popular because it lets you build UI in small pieces, reuse those pieces, and manage changing data in a predictable way. It is maintained by Meta and widely used in dashboards, admin panels, SPAs, e-commerce apps, and mobile apps through React Native.

Component Based

Split complex UI into reusable components like buttons, cards, forms, and layouts.

Reactive UI

When state changes, React recalculates the UI and updates only what changed.

JSX Syntax

Write HTML-like UI together with JavaScript logic in one place.

Huge Ecosystem

Works with Vite, Next.js, React Router, Zustand, Redux, testing tools, and more.

Think of React like this
HTML + JS + CSS

Traditional pages often mix DOM selection, manual updates, and event handling all over the place.

React

You define UI as components and let React manage re-rendering when data changes.

Virtual DOM & Reconciliation

The Virtual DOM is a lightweight JavaScript copy of the real DOM that React maintains in memory. When state changes, React compares the new Virtual DOM with the previous one (called diffing) and applies only the minimum required changes to the real DOM — this process is called Reconciliation.

State Changes

You call setState or a hook updates. React creates a new Virtual DOM tree.

Diffing (Reconciliation)

React compares the new Virtual DOM tree with the previous snapshot and finds what changed.

Minimal DOM Update

Only the changed nodes are updated in the real DOM. This makes React very fast.

Why Keys Matter in Reconciliation
❌ Bad — index as key
{items.map((item, index) => (
  <li key={index}>{item}</li>  // React gets confused on reorder
))}
✔ Good — stable unique ID
{items.map((item) => (
  <li key={item.id}>{item.name}</li>  // React tracks correctly
))}
ConceptDescription
Virtual DOMIn-memory JS object tree — cheap to create and compare
DiffingAlgorithm that finds minimum changes between old and new Virtual DOM
ReconciliationProcess of applying those minimal changes to the real DOM
FiberReact 16+ internal engine that makes reconciliation interruptible (enables Concurrent Mode)
KeysHelp React match old and new elements in lists during diffing

React StrictMode

StrictMode is a development-only wrapper that intentionally double-invokes certain functions to help you detect side effects, deprecated APIs, and potential bugs early. It has zero impact on production builds.

1How to Enable
// main.jsx — wraps your entire app
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import App from './App';

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>
);
2What StrictMode Detects
Double-invokes render

Renders components twice in dev to detect side effects in the render phase (render itself must be pure).

Double-fires effects

Runs useEffect setup + cleanup + setup again to verify your cleanup is correct.

Deprecated API warnings

Warns about legacy React APIs and patterns that will be removed in future versions.

Production safe

StrictMode is stripped out in production builds — your app runs normally for users.

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };

2useFetch — Reusable API Fetching
import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const controller = new AbortController();

    setLoading(true);
    fetch(url, { signal: controller.signal })
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then(setData)
      .catch((err) => {
        if (err.name !== 'AbortError') setError(err.message);
      })
      .finally(() => setLoading(false));

    return () => controller.abort();
  }, [url]);

  return { data, loading, error };
}

// Usage — replaces repetitive useEffect + useState boilerplate
function Users() {
  const { data, loading, error } = useFetch('https://jsonplaceholder.typicode.com/users');

  if (loading) return <p>Loading...</p>;
  if (error) return <p className="text-danger">{error}</p>;

  return (
    <ul>
      {data.map((u) => <li key={u.id}>{u.name}</li>)}
    </ul>
  );
}
3useLocalStorage — Persistent State
import { useState } from 'react';

function useLocalStorage(key, initialValue) {
  const [stored, setStored] = useState(() => {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  function setValue(value) {
    const valueToStore = value instanceof Function ? value(stored) : value;
    setStored(valueToStore);
    localStorage.setItem(key, JSON.stringify(valueToStore));
  }

  return [stored, setValue];
}

// Usage — survives page refresh
function Settings() {
  const [theme, setTheme] = useLocalStorage('theme', 'dark');

  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      Current theme: {theme}
    </button>
  );
}
4useDebounce — Delay Fast Updates
import { useState, useEffect } from 'react';

function useDebounce(value, delay = 500) {
  const [debounced, setDebounced] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer); // reset timer on every value change
  }, [value, delay]);

  return debounced;
}

// Usage — API call fires only 500ms after user stops typing
function SearchPage() {
  const [query, setQuery] = useState('');
  const debouncedQuery = useDebounce(query, 500);

  useEffect(() => {
    if (debouncedQuery) {
      fetchResults(debouncedQuery); // only calls API when user pauses
    }
  }, [debouncedQuery]);

  return <input value={query} onChange={(e) => setQuery(e.target.value)} placeholder="Search..." />;
}

3AbortController — Cleanup on Unmount

When a component unmounts before a fetch completes (e.g. user navigates away), you must cancel the request to avoid setting state on an unmounted component.

import { useEffect, useState } from 'react';

function Posts() {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    const controller = new AbortController(); // create controller

    fetch('https://jsonplaceholder.typicode.com/posts', {
      signal: controller.signal,              // attach signal to fetch
    })
      .then((res) => res.json())
      .then(setPosts)
      .catch((err) => {
        if (err.name === 'AbortError') return; // ignore intentional abort
        console.error(err);
      });

    return () => controller.abort(); // cancel fetch when component unmounts
  }, []);

  return posts.map((p) => <p key={p.id}>{p.title}</p>);

2List Virtualization

Rendering thousands of DOM nodes is slow. Virtualization renders only the rows visible in the viewport — drastically improving performance for large lists.

npm install @tanstack/react-virtual
import { useVirtualizer } from '@tanstack/react-virtual';
import { useRef } from 'react';

const allItems = Array.from({ length: 10000 }, (_, i) => `Item ${i + 1}`);

function VirtualList() {
  const parentRef = useRef(null);

  const virtualizer = useVirtualizer({
    count: allItems.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 40,  // estimated row height in px
  });

  return (
    <div ref={parentRef} style={{ height: '400px', overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize() + 'px', position: 'relative' }}>
        {virtualizer.getVirtualItems().map((virtualRow) => (
          <div
            key={virtualRow.index}
            style={{
              position: 'absolute',
              top: virtualRow.start + 'px',
              height: virtualRow.size + 'px',
              width: '100%',
            }}
          >
            {allItems[virtualRow.index]}
          </div>
        ))}
      </div>
    </div>
  );
}

React 18 introduced Concurrent Mode — React can now pause, resume, and interrupt rendering to keep the UI responsive. These hooks let you mark certain state updates as lower priority.

1useTransition — Non-Urgent Updates

Use useTransition to mark state updates as non-urgent so React can keep the UI responsive during slow re-renders (e.g., filtering a large list).

import { useState, useTransition } from 'react';

function SearchResults({ query }) {
  // Imagine 10,000 items being filtered here
  const results = hugeList.filter(item => item.includes(query));
  return results.map(r => <p key={r}>{r}</p>);
}

function SearchPage() {
  const [query, setQuery] = useState('');
  const [deferredQuery, setDeferredQuery] = useState('');
  const [isPending, startTransition] = useTransition();

  function handleChange(e) {
    setQuery(e.target.value); // urgent update — input responds instantly

    startTransition(() => {
      setDeferredQuery(e.target.value); // non-urgent — list updates when free
    });
  }

  return (
    <>
      <input value={query} onChange={handleChange} placeholder="Search..." />
      {isPending && <p className="text-muted">Updating results...</p>}
      <SearchResults query={deferredQuery} />
    </>
  );
}

Best Practices

Component Design
  • Keep components small and focused on one job.
  • Name components with PascalCase: UserCard, not userCard.
  • Name hooks with use prefix: useFetch, not fetchData.
  • Use named exports for components, default for pages.
  • Prefer composition over deeply nested prop drilling.
State & Data
  • Never mutate state directly — always return new values.
  • Use stable unique keys (not index) in lists.
  • Keep state as close to where it's used as possible.
  • Use TanStack Query for server state, Zustand for client global state.
  • Clean up effects: timers, subscriptions, AbortControllers.
Performance
  • Measure before optimizing — use React DevTools Profiler.
  • Use React.memo only for expensive components that re-render often.
  • Virtualize large lists with @tanstack/react-virtual.
  • Use lazy() + Suspense to split large page bundles.
  • Use useTransition for non-urgent UI updates.
Security & Quality
  • Never store secrets in VITE_ env vars — they're public.
  • Sanitize any user content rendered as HTML (avoid dangerouslySetInnerHTML).
  • Validate all form inputs and API responses.
  • Use ESLint + Prettier for consistent code style.
  • Use TypeScript for type safety in team projects.

Environment Variables

Environment variables let you store configuration like API URLs and keys outside your source code. In Vite-based React projects, variables must start with VITE_ to be exposed to the browser.

1Create .env Files
# .env  — loaded in all environments
VITE_API_URL=https://api.example.com
VITE_APP_NAME=MyApp

# .env.development  — only in dev (npm run dev)
VITE_API_URL=http://localhost:8000

# .env.production  — only in prod (npm run build)
VITE_API_URL=https://api.example.com
2Read in Code
// Access via import.meta.env in Vite
const apiUrl = import.meta.env.VITE_API_URL;
const appName = import.meta.env.VITE_APP_NAME;

async function getUsers() {
  const res = await fetch(`${apiUrl}/users`);
  return res.json();
}

function App() {
  return <h1>{appName}</h1>;
}

Accessibility (a11y)

Accessibility ensures your React app can be used by everyone — including people using screen readers, keyboard-only navigation, or assistive technology. React renders standard HTML, so standard HTML accessibility rules apply.

1Semantic HTML
❌ Non-semantic
<div onClick={submit}>Submit</div>
<div className="nav">...</div>
<div className="header">...</div>
✔ Semantic
<button onClick={submit}>Submit</button>
<nav>...</nav>
<header>...</header>
2ARIA Attributes
// aria-label — describes element when visible text is absent
<button aria-label="Close modal"><i className="bi bi-x" /></button>

// aria-hidden — hides decorative elements from screen readers
<i className="bi bi-heart" aria-hidden="true" />
<span>Like</span>

// aria-live — announces dynamic content changes
<div aria-live="polite">
  {loading ? 'Loading...' : `${results.length} results found`}
</div>

// aria-expanded — for accordion/dropdown state
<button aria-expanded={isOpen} onClick={toggle}>
  {isOpen ? 'Collapse' : 'Expand'}
</button>

// role — when semantic tag isn't appropriate
<div role="alert" className="error">{errorMessage}</div>
3Form Labels & Focus Management
import { useId, useRef, useEffect } from 'react';

function AccessibleForm() {
  const emailId = useId();
  const modalRef = useRef(null);

  // Move focus to modal when it opens
  useEffect(() => {
    modalRef.current?.focus();
  }, []);

  return (
    <form>
      {/* Always link labels to inputs */}
      <label htmlFor={emailId}>Email address</label>
      <input
        id={emailId}
        type="email"
        aria-required="true"
        aria-describedby="email-hint"
      />
      <p id="email-hint" className="text-muted small">We'll never share your email.</p>

      {/* Skip link for keyboard users */}
      <a href="#main-content" className="visually-hidden-focusable">
        Skip to main content
      </a>

      {/* Focus trap container (tabIndex makes div focusable) */}
      <div ref={modalRef} tabIndex={-1}>
        Modal content here
      </div>
    </form>
  );
}
4Keyboard Navigation
// Handle Enter/Space key on custom interactive elements
function CustomButton({ onClick, children }) {
  function handleKeyDown(e) {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      onClick();
    }
  }

  return (
    <div
      role="button"
      tabIndex={0}
      onClick={onClick}
      onKeyDown={handleKeyDown}
    >
      {children}
    </div>
  );
}
RuleWhy
Use semantic HTML tagsScreen readers understand <button>, <nav>, <main> correctly
Always link <label> to inputsScreen readers announce the label when input is focused
Provide alt text for imagesDescribes images to visually impaired users
Ensure color contrast ratio ≥ 4.5:1Text readable by people with low vision
Never remove focus outlineKeyboard users need to see which element is focused
Manage focus when modals openScreen reader cursor should move into the modal

Animations with Framer Motion

Framer Motion is the most popular React animation library. It lets you add smooth animations, transitions, layout animations, and gestures with a declarative API — no CSS keyframes needed.

1Install
npm install motion
2Basic Animate
import { motion } from 'motion/react';

// Fade in + slide up on mount
function Card() {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: 0.4 }}
      className="card p-4"
    >
      Hello, I animated in!
    </motion.div>
  );
}
3Exit Animations with AnimatePresence
import { motion, AnimatePresence } from 'motion/react';
import { useState } from 'react';

function Toast({ message }) {
  const [visible, setVisible] = useState(true);

  return (
    <AnimatePresence>
      {visible && (
        <motion.div
          initial={{ opacity: 0, x: 50 }}
          animate={{ opacity: 1, x: 0 }}
          exit={{ opacity: 0, x: 50 }}   {/* plays on unmount */}
          transition={{ duration: 0.3 }}
          className="alert alert-success"
        >
          {message}
          <button onClick={() => setVisible(false)}>×</button>
        </motion.div>
      )}
    </AnimatePresence>
  );
}
4Hover & Tap Gestures
import { motion } from 'motion/react';

function AnimatedButton({ children, onClick }) {
  return (
    <motion.button
      whileHover={{ scale: 1.05, backgroundColor: '#0ea5e9' }}
      whileTap={{ scale: 0.95 }}
      transition={{ type: 'spring', stiffness: 400 }}
      onClick={onClick}
      className="btn btn-primary"
    >
      {children}
    </motion.button>
  );
}
5Layout Animations
import { motion } from 'motion/react';
import { useState } from 'react';

function ExpandableCard() {
  const [expanded, setExpanded] = useState(false);

  return (
    <motion.div
      layout  // automatically animates layout changes
      onClick={() => setExpanded(!expanded)}
      style={{ cursor: 'pointer', padding: '16px', borderRadius: '12px', background: '#1e293b' }}
    >
      <motion.h4 layout>Click to {expanded ? 'collapse' : 'expand'}</motion.h4>
      {expanded && (
        <motion.p
          initial={{ opacity: 0 }}
          animate={{ opacity: 1 }}
        >
          Extra content revealed with smooth layout animation.
        </motion.p>
      )}
    </motion.div>
  );
}

React Native — Mobile with React

React Native lets you build native iOS and Android apps using React and JavaScript. You use the same component model, hooks, and state management — but instead of HTML tags, you use React Native's built-in components (View, Text, TouchableOpacity).

iOS & Android

One codebase for both platforms using native components.

Same React Skills

Hooks, state, props, context — all the same as React web.
